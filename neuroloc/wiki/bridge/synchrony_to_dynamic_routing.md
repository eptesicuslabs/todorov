# bridge: synchrony to dynamic routing

status: current (as of 2026-04-23).

## translation

the strongest reusable lesson from synchrony and thalamic-routing work is not “copy oscillations.” it is “treat effective communication as state-dependent.” the graph can stay fixed while the active subgraph changes.

## implications

- routing should be an explicit architecture concern
- access control may matter as much as storage quality
- oscillatory or phase-like abstractions are candidates only insofar as they implement state-dependent routing or access

## what remains open

- whether the project ever needs a direct phase-like mechanism
- whether routing should be a separate control path or emerge inside memory and state paths
