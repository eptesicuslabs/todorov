# bridge: indexing to memory interfaces

status: current (as of 2026-04-23).

## translation

hippocampal indexing theory and bounded-memory ai systems converge on one useful interface idea:

- store a compact handle
- use that handle to drive reconstruction, completion, or targeted retrieval later

this suggests that future project memory interfaces should be judged by how well they support compact addressing plus useful reinstatement, not only by how much dense content they can store directly.

## implications

- address quality matters as much as storage capacity
- recall should be tested under degraded cues and one-to-many binding
- replay and reinstatement should be treated as interface clients of the index, not as unrelated extras

## what remains open

- whether the best index is slot-like, key-value-like, object-like, or world-state-like
- how much of reinstatement should be handled by a learned generative model versus an explicit memory store
