# path integration

status: definitional. last fact-checked 2026-04-16.

**why this matters**: path integration is the brain's recurrent state update -- accumulating velocity signals to track position over time -- and its mathematical form (x_{t+1} = x_t + v_t * dt) is structurally identical to the state update in linear recurrent layers like Mamba and KDA.

## summary

**path integration** (also called **dead reckoning** -- estimating current position by integrating displacement from a known starting point) is the computational process by which the brain estimates its current position by integrating self-motion signals -- velocity and heading direction -- over time. the basic computation is a continuous update: x_{t+1} = x_t + v_t * dt, where x is position, v is the velocity vector (speed times heading direction), and dt is the time step. path integration allows navigation in the absence of external landmarks (e.g., in darkness), but accumulates error over time and must be periodically corrected by landmark-based recalibration.

path integration is believed to be implemented by the medial entorhinal cortex (MEC) through the coordinated activity of head direction cells (providing heading), speed cells (providing speed magnitude), and grid cells (encoding the integrated position estimate). the hippocampal place cell map (see [[place_cells]]) is downstream of this computation.

## the computational problem

an animal needs to know where it is. two strategies are available:

1. **landmark-based localization**: determine position from recognized features of the environment (visual landmarks, sounds, odors). this is precise but requires known landmarks
2. **path integration**: estimate position by tracking how far and in what direction one has traveled since the last known location. this works in novel or featureless environments but accumulates error

in practice, animals use both strategies simultaneously. path integration provides a continuous position estimate that is periodically corrected when a recognized landmark is encountered. this is formally equivalent to a **Kalman filter** (a recursive algorithm that combines noisy predictions with noisy measurements to estimate the true state). path integration provides the prediction step (state propagation via the motion model), and landmark recognition provides the correction step (measurement update).

ML analog: this prediction-correction loop is analogous to the hybrid architecture of KDA + MLA in todorov. KDA accumulates state over time (path integration / prediction), while MLA retrieves specific past tokens (landmark recognition / correction).

the mathematical formulation for 2D path integration:

    x(t) = x(0) + integral from 0 to t of v(tau) * cos(theta(tau)) d(tau)
    y(t) = y(0) + integral from 0 to t of v(tau) * sin(theta(tau)) d(tau)

where v(tau) is running speed at time tau and theta(tau) is heading direction. in discrete form:

    x_{t+1} = x_t + v_t * cos(theta_t) * dt
    y_{t+1} = y_t + v_t * sin(theta_t) * dt

this requires two inputs: a speed signal v_t and a direction signal theta_t. the brain has dedicated cell types for each.

## biological implementation

### head direction cells

**head direction (HD) cells** fire when the animal faces a specific direction in the horizontal plane, regardless of the animal's location, behavior, or body position. discovered by Taube, Muller, and Ranck (1990) in the postsubiculum, they are found throughout the limbic circuit: anterodorsal thalamic nucleus (ADN), lateral mammillary nucleus (LMN), dorsal tegmental nucleus (DTN), retrosplenial cortex, MEC, and presubiculum.

properties:
- each HD cell has a preferred firing direction (PFD): the direction at which it fires maximally
- the tuning curve is approximately Gaussian, with a half-width of ~60-90 degrees
- the population of HD cells collectively covers all 360 degrees
- the preferred direction is allocentric (relative to the environment, not the body)
- HD cell firing is updated by vestibular input (angular velocity from the semicircular canals) and can persist in the absence of visual landmarks

the HD signal is generated by a **ring attractor network** (a recurrent network whose stable states form a ring, with a single bump of activity that can slide around the ring). the current heading is represented by a bump of activity at the corresponding position on a ring of HD cells.

ML analog: the ring attractor is analogous to rotary positional embeddings (RoPE) in transformers. both encode a continuous angular variable (heading direction / position) using a rotation in a periodic space. angular velocity signals from the vestibular system shift the bump around the ring: clockwise rotation shifts the bump clockwise, and vice versa. the ring attractor maintains a coherent heading estimate through recurrent excitation within the bump and lateral inhibition between the bump and the rest of the ring.

the circuit: angular velocity signals originate in the DTN (which receives from the vestibular nuclei), project to the LMN (where angular velocity is combined with the current HD signal), and from there to the ADN (which projects to cortex). the ADN is the primary thalamic relay for the HD signal. from cortex, the HD signal reaches MEC, where it is combined with speed information for path integration.

HD cells are anchored to visual landmarks when available: rotating a prominent cue card in an otherwise featureless environment causes HD cell preferred directions to rotate by the same amount (Taube et al. 1990). in the absence of landmarks, HD cells maintain their relative orientations but can drift slowly from their anchored positions, accumulating angular error at ~1-5 degrees per minute (Knierim, Kudrimoti & McNaughton 1998).

### speed cells

**speed cells** in the MEC fire at rates that increase linearly with the animal's running speed. discovered by Kropff, Carmichael, Moser, and Moser (2015), they are a functionally distinct population from grid cells, head direction cells, and border cells.

properties:
- firing rate = baseline + gain * speed, with a strong linear correlation (r > 0.5 for the majority of speed cells)
- the speed signal is context-invariant: the same cell has the same speed-rate relationship across different environments
- speed cells share with grid cells a prospective temporal bias of ~50-80 ms: their firing reflects the speed the animal WILL be running in 50-80 ms, not the current speed. this anticipation may compensate for neural processing delays
- speed cells are found predominantly in deeper layers of MEC (layers III, V, VI)

speed cells provide the magnitude component of the velocity vector needed for path integration. their linear encoding means that the firing rate can be directly multiplied by the heading direction (from HD cells) to compute the velocity vector.

### conjunctive cells

many neurons in MEC encode combinations of spatial variables:
- **grid x HD cells**: fire at grid vertices but only when the animal faces a particular direction. these are found in deeper MEC layers and may carry the velocity signal needed to update the grid cell attractor
- **grid x speed cells**: fire at grid vertices with rate modulated by speed
- **HD x speed cells**: firing direction modulated by speed

conjunctive cells are particularly important for path integration because the velocity signal (speed x direction) must be decomposed into directional components to shift the grid cell attractor along the correct axis. grid x HD conjunctive cells may implement this decomposition: each conjunctive cell combines "where I am in the grid" with "which direction I'm moving" to generate the appropriate shift signal.

## error accumulation and correction

path integration is inherently noisy. errors accumulate because:
- speed estimation is noisy (both neural noise and sensory noise)
- direction estimation is noisy (vestibular signals have limited precision)
- integration itself accumulates errors: a small systematic bias in speed or direction produces a linearly growing position error

empirical measurements of path integration error in rodents:
- in darkness, grid cell firing patterns drift from their light-on positions at ~1-2 cm per meter of distance traveled (Hardcastle et al. 2015)
- head direction cells drift at ~1-5 degrees per minute of darkness (Stackman & Taube 1998; Knierim et al. 1998)
- after extended periods without visual landmarks (~10-30 minutes), the spatial representation degrades substantially

error correction mechanisms:
1. **visual landmarks**: when a recognized landmark is encountered, the position estimate is corrected by "resetting" the grid cell/place cell activity to the known position associated with that landmark. this reset is fast (~seconds) and can be partial (different modules may reset independently)
2. **boundary information**: border cells fire at environmental edges and provide geometric constraints that can partially correct path integration error even without discrete landmarks
3. **self-consistency between modules**: because grid cell modules have incommensurate spacings (ratio ~1.4), the combined code is redundant. errors that shift one module's representation are inconsistent with other modules, potentially enabling error detection and correction through inter-module interactions

the error correction process is analogous to SLAM (simultaneous localization and mapping) in robotics: the agent maintains a running position estimate via path integration (odometry) and corrects it when recognizable features are encountered (loop closure).

## computational models

### attractor-based path integration

in continuous attractor network (CAN) models of grid cells (see [[grid_cells]]), path integration is implemented by velocity-driven translation of the activity pattern:
1. the grid cell network has a stable hexagonal pattern of activity bumps
2. conjunctive grid x HD cells receive directional velocity signals
3. the velocity signal creates an asymmetry in the recurrent weights, causing the activity pattern to shift in the direction of movement at a rate proportional to speed
4. the shifted pattern encodes the new position

Burak and Fiete (2009) showed that CAN models can perform accurate path integration over extended trajectories, with error growth that matches experimental data.

### oscillatory path integration

in oscillatory interference (OI) models, path integration is performed by velocity-controlled oscillators (VCOs):
1. each VCO has a frequency that increases from a baseline (theta, ~8 Hz) proportionally to the component of velocity along a preferred direction
2. the phase of each VCO relative to the baseline encodes displacement along that direction: phi = (2*pi*d)/s, where d is displacement and s is the grid spacing
3. position is encoded in the pattern of phases across VCOs

the OI model naturally produces path integration because velocity is integrated in the phase of each oscillator. however, phase noise in biological oscillators limits the accuracy and duration of path integration without reset.

### vector-based navigation

Banino et al. (2018) trained a recurrent neural network to perform path integration (predicting position from velocity sequences) and found that grid-like representations emerged spontaneously in the hidden units. when these grid-like units were used as the spatial representation for a reinforcement learning agent, the agent performed vector-based navigation (shortcuts, novel routes) that rivaled human performance.

this result demonstrates that grid-like representations are not just an empirical curiosity of the MEC but arise as computationally optimal solutions to the path integration problem. the connection to todorov: if gradient descent discovers grid-like representations when path integration is the task, it might discover PGA-like geometric operations when spatial reasoning is needed. but todorov's task is language modeling, not path integration.

## relationship to todorov

the path integration computation -- x_{t+1} = x_t + v_t * dt -- is structurally similar to the recurrent state update in KDA:

    S_{t+1} = alpha * S_{t} + beta_t * k_t * v_t^T

both are first-order recurrences that accumulate information over time. the differences are substantial:
- path integration integrates a 2D velocity vector; KDA accumulates rank-1 outer products
- path integration has no forgetting (or minimal, error-driven forgetting); KDA has exponential decay via alpha
- path integration error is corrected by landmarks; KDA has no error correction mechanism
- path integration operates in physical space; KDA operates in feature space

the PGA module could in principle compute translations (via projective points) and rotations (via rotors), which are the geometric operations underlying path integration. but in todorov's implementation, the GP is a self-interaction between two projections of the same input x, with no temporal integration and no velocity signal. see [[spatial_computation_to_pga]] for the full analysis.

## challenges

path integration faces several fundamental limitations. first, error accumulation is unavoidable and grows with distance traveled. biological path integration drifts at ~1-2 cm per meter, which means that after 100 meters the estimate can be off by 1-2 meters. this makes path integration unreliable for long-distance navigation without landmark correction. the brain's solution (periodic landmark-based reset) requires prior knowledge of the environment, limiting the utility of path integration in truly novel settings.

second, the neural mechanism that converts velocity signals into pattern shifts in the grid cell network is not directly observed. the CAN model predicts velocity-driven asymmetric connectivity, but no experiment has measured the specific synaptic connections that implement this asymmetry. the mechanism remains a computational hypothesis, not an established circuit.

third, path integration models assume accurate velocity input, but the biological speed signal (from speed cells) has noise, latency (~50-80 ms prospective bias), and potential systematic biases. how the system compensates for these input imperfections is unclear, and whether the prospective bias is a bug (processing delay) or a feature (predictive compensation) is debated.
