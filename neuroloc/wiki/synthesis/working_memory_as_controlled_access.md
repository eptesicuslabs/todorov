# working memory as controlled access

status: current (as of 2026-04-23).

## thesis

working memory is better modeled as controlled access to a small focus plus activated long-term structure than as one sealed buffer. the key computational story is:

- choose what gets privileged
- keep the currently needed subset available
- rebind or retrieve what was not already in focus

that makes working memory a routing and control problem as much as a storage problem.

## what looks established

- the focus of attention is small and selective
- working memory performance depends strongly on control operations, not just raw storage
- long-term memory and episodic structure can become temporarily available for controlled use
- replay, reinstatement, and indexing blur the line between “working” and “long-term” memory

## what this implies

future project language should avoid treating memory as one capacity number. it should distinguish:

- current focus
- stored state
- retrieval path
- control path

that fits the existing phase-1 move away from passkey-only gating and toward state plus action metrics.

## what remains speculative

- a full workspace-plus-self-model architecture
- narrative-self stories as immediate project guidance

the useful part is the control abstraction, not the philosophy.
